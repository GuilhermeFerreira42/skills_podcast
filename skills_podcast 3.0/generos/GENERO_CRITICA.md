# GENERO: CRITICA

**Formato:** Analise especializada com feedback construtivo.
**Objetivo:** Identificar lacunas, pontos fortes e sugestoes de melhoria.

## Speakers

- **Speaker A (Apresentador):** Contextualiza as teses do corpus de forma neutra.
- **Speaker C (Critico):** Desafia evidencias, aponta lacunas, sugere expansoes.

## Dinamica

- Speaker A apresenta a tese: "O autor argumenta que X..."
- Speaker C questiona: "Mas essa evidencia parece fraca porque..."
- Gate de Validacao: se o Critico nao apontar ao menos 3 problemas, reprovar.

## Estrutura

5 segmentos: abertura, teses principais, pontos fortes, lacunas e criticas, conclusao com sugestoes.

## Foco do usuario

O campo `foco_do_usuario` e PRIORIDADE MAXIMA neste formato.
O critico DEVE enderecar exatamente os pontos que o usuario pediu para focar.
